## 레벨 시스템

### 레벨업 공식
```
필요 경험치 = 레벨 × 100
```

### 레벨별 필요 경험치

| 레벨 | 필요 경험치 | 누적 경험치 | 타이틀 |
|------|-------------|-------------|--------|
| 1 → 2 | 100 | 100 | 초보 학습자 |
| 2 → 3 | 200 | 300 | 초보 학습자 |
| 3 → 4 | 300 | 600 | 초보 학습자 |
| 4 → 5 | 400 | 1,000 | 초보 학습자 |
| 5 → 6 | 500 | 1,500 | 열심 학생 |
| 6 → 7 | 600 | 2,100 | 열심 학생 |
| 7 → 8 | 700 | 2,800 | 열심 학생 |
| 8 → 9 | 800 | 3,600 | 열심 학생 |
| 9 → 10 | 900 | 4,500 | 열심 학생 |
| 10 → 11 | 1,000 | 5,500 | 학자 |
| 15 → 16 | 1,500 | 12,000 | 박사 |
| 20 → 21 | 2,000 | 21,000 | 석학 |
| 30 → 31 | 3,000 | 46,500 | 대학자 |
| 50+ | - | - | 전설의 학자 |

### 레벨 타이틀

| 레벨 범위 | 타이틀 |
|-----------|--------|
| 1 ~ 4 | 초보 학습자 |
| 5 ~ 9 | 열심 학생 |
| 10 ~ 14 | 학자 |
| 15 ~ 19 | 박사 |
| 20 ~ 29 | 석학 |
| 30 ~ 49 | 대학자 |
| 50+ | 전설의 학자 |

### 경험치 획득

| 행동 | 경험치 |
|------|--------|
| 게임 완료 (기본) | 50 EXP |
| 정답 1개당 | +10 EXP |
| **5문제 전부 정답** | **100 EXP** |
| 5문제 중 3개 정답 | 80 EXP |
| 5문제 중 0개 정답 | 50 EXP |

## 에너지 시스템

### 에너지 공식
```
에너지 = 연속 출석 일수 × 5 (최대 100)
```

### 연속 출석별 에너지

| 연속 출석 | 에너지 | 포인트 보너스 |
|-----------|--------|---------------|
| 0일 (첫날) | 0 | ×1.0 (보너스 없음) |
| 1일 | 5 | ×1.05 |
| 2일 | 10 | ×1.1 |
| 5일 | 25 | ×1.25 |
| 10일 | 50 | ×1.5 |
| 15일 | 75 | ×1.75 |
| 20일+ | 100 (최대) | ×2.0 (최대) |

### 에너지 보너스 공식
```
포인트 보너스 배율 = 1 + (에너지 / 100)
```

### 에너지 리셋 조건
- **하루라도 게임을 안하면** → 연속 출석 0, 에너지 0으로 리셋
- 레벨과 포인트는 유지됨

## 포인트 시스템

### 포인트 획득

| 행동 | 기본 포인트 | 에너지 100일 때 |
|------|-------------|-----------------|
| 문제 정답 | 10 P | 20 P |
| 게임 완료 보너스 | 50 P | 50 P (보너스 미적용) |
| 일일 퀘스트 완료 | 300 P | 300 P |
| 주간 퀘스트 완료 | 1,000 P | 1,000 P |

### 게임 1회 완료 시 포인트 (5문제 기준)

| 정답 수 | 에너지 0 | 에너지 50 | 에너지 100 |
|---------|----------|-----------|------------|
| 5/5 | 100 P | 125 P | 150 P |
| 4/5 | 90 P | 110 P | 130 P |
| 3/5 | 80 P | 95 P | 110 P |
| 0/5 | 50 P | 50 P | 50 P |

※ 게임 완료 보너스 50P는 에너지 보너스 미적용

## 퀘스트 시스템

### 일일 퀘스트

| 조건 | 보상 |
|------|------|
| 수학 + 영어 + 한자 각 1회 이상 완료 | 300 P |

- 매일 자정에 리셋
- 한 번만 수령 가능

### 주간 퀘스트

| 조건 | 보상 |
|------|------|
| 일주일 동안 총 21회 게임 완료 | 1,000 P |

- 매주 리셋
- 하루 평균 3회 (수학 + 영어 + 한자)

## 미니게임 비용

|-------------|-------------|
| 30분 | 600 P |
| 1시간 | 1,000 P |

```
플레이 가능 시간(분) = 보유 포인트 / 17 (최대 120분)
```

## 하루 플레이 시뮬레이션

### 시나리오: 에너지 0 (첫날)

| 게임 | 정답 | 포인트 | 경험치 |
|------|------|--------|--------|
| 수학 5/5 | 5개 | 100 P | 100 EXP |
| 영어 5/5 | 5개 | 100 P | 100 EXP |
| 한자 5/5 | 5개 | 100 P | 100 EXP |
| 일일 퀘스트 | - | 300 P | - |
| **합계** | - | **600 P** | **300 EXP** |

→ 약 **35분** 미니게임 가능

### 시나리오: 에너지 100 (20일 연속)

| 게임 | 정답 | 포인트 | 경험치 |
|------|------|--------|--------|
| 수학 5/5 | 5개 | 150 P | 100 EXP |
| 영어 5/5 | 5개 | 150 P | 100 EXP |
| 한자 5/5 | 5개 | 150 P | 100 EXP |
| 일일 퀘스트 | - | 300 P | - |
| **합계** | - | **750 P** | **300 EXP** |

→ 약 **44분** 미니게임 가능

### 시나리오: 에너지 100 + 주간 퀘스트

| 항목 | 포인트 |
|------|--------|
| 일일 (수학+영어+한자) | 750 P |
| 주간 퀘스트 | 1,000 P |
| **합계** | **1,750 P** |

→ 약 **1시간 43분** 미니게임 가능

## 밸런스 조정 포인트

수정이 필요하면 `RewardRepository.kt`에서 다음 값들을 조정:

```kotlin
// 에너지 증가량 (연속 출석 1일당)
val newEnergy = min(_currentStreak.value * 5, 100)  // 5 → 조정

// 레벨업 필요 경험치
fun getRequiredExp(level: Int): Int = level * 100  // 100 → 조정

// 게임 완료 경험치
val expGained = 50 + (correctAnswers * 10)  // 50, 10 → 조정

// 게임 완료 보너스 포인트
_totalPoints.update { it + 50 }  // 50 → 조정

// 일일 퀘스트 보상
_totalPoints.update { it + 300 }  // 300 → 조정

// 주간 퀘스트 보상
_totalPoints.update { it + 1000 }  // 1000 → 조정
```
